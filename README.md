# Modeus Calendar

Сервис подписки на расписание ТюмГУ в формате iCalendar.

Пользователь вводит ФИО и инвайт-код → получает личную ссылку → добавляет в Apple Calendar / Google Calendar → расписание обновляется автоматически каждые 3 часа.

## Как это работает

1. Администратор создаёт инвайт-коды через Telegram-бота
2. Пользователь открывает сайт, вводит ФИО и инвайт-код
3. Если найдено несколько человек с таким ФИО — появляется окно выбора с направлениями
4. Сервис создаёт подписку и выдаёт уникальную ICS-ссылку
5. Пользователь добавляет ссылку в Apple Calendar или Google Calendar
6. Расписание загружается из Modeus и обновляется автоматически каждые 3 часа

## Архитектура

```
Браузер / Apple Calendar / Google Calendar
         │
         ▼
    frontend (nginx:80)
         │
         ├── /api/*    ──────► backend (Node.js :3000)
         └── /<token>  ──────►    │
                                  ├── SQLite (volume calendar-data)
                                  └── cron → Modeus API (utmn.modeus.org)

    bot (Node.js)
         ├── читает SQLite напрямую (статистика, списки)
         └── /api/internal/register ──► backend (внутренняя сеть Docker)
```

**Три Docker-контейнера:**

| Контейнер  | Образ         | Назначение                                         |
|------------|---------------|----------------------------------------------------|
| `frontend` | nginx:alpine  | React-билд + реверс-прокси на backend              |
| `backend`  | node:22-alpine| API, ICS-фиды, cron-синхронизация с Modeus         |
| `bot`      | node:22-alpine| Telegram-бот для администрирования                 |

## Быстрый старт (Docker)

### Требования

- Docker + Docker Compose (Docker Desktop или `docker-ce`)
- Аккаунт на `utmn.modeus.org` (логин в формате `ivanov.ii@s.utmn.ru`)
- Telegram-бот (токен от [@BotFather](https://t.me/BotFather))

### Шаг 1 — Скопируй проект на сервер

```bash
# Через git:
git clone <repo-url> /root/app
cd /root/app

# Или через rsync с локальной машины:
rsync -av \
  --exclude='node_modules' --exclude='client/node_modules' --exclude='bot/node_modules' \
  --exclude='dist' --exclude='client/dist' --exclude='bot/dist' \
  --exclude='.tokens.json' --exclude='*.db' --exclude='.env' \
  ./modeus-calendar/ root@<IP>:/root/app/
```

### Шаг 2 — Создай `.env`

```bash
cp /root/app/.env.example /root/app/.env
nano /root/app/.env
```

Заполни файл (подробнее в разделе [Переменные окружения](#переменные-окружения)):

```env
MODEUS_USERNAME=ivanov.ii@s.utmn.ru
MODEUS_PASSWORD=ВашПароль

PORT=3000
DB_PATH=/app/data/calendar.db
CRON_SCHEDULE=0 */3 * * *
SYNC_WEEKS_AHEAD=4

INVITE_REQUIRED=true
INTERNAL_SECRET=придумай-любую-длинную-строку

TELEGRAM_BOT_TOKEN=1234567890:ABCDEFghijklmnop
TELEGRAM_ADMIN_ID=123456789
```

> **Где взять `TELEGRAM_ADMIN_ID`?**
> Напиши боту [@userinfobot](https://t.me/userinfobot) — он ответит твоим ID.

> **Как создать бота и получить токен?**
> Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → следуй инструкции → скопируй токен.

### Шаг 3 — Запусти

```bash
cd /root/app
docker compose up -d --build
```

Сборка занимает 2–5 минут (компилируется TypeScript и нативные зависимости). После этого:

- Сайт доступен на `http://<IP>:3000`
- Telegram-бот активен, напиши ему `/start`

### Полезные команды

```bash
# Посмотреть статус контейнеров
docker compose ps

# Логи всех контейнеров (Ctrl+C для выхода)
docker compose logs -f

# Логи только бэкенда
docker compose logs -f backend

# Логи только бота
docker compose logs -f bot

# Перезапустить после обновления кода
git pull
docker compose up -d --build

# Остановить (данные сохраняются)
docker compose down

# Остановить и удалить все данные (включая базу!)
docker compose down -v
```

> Данные SQLite хранятся в Docker volume `calendar-data` и **не удаляются** при `docker compose down`.

## Telegram-бот

Бот — основной инструмент администрирования. Через него выдаются инвайт-коды, просматривается статистика и регистрируются пользователи напрямую, минуя сайт.

Бот работает **только для одного администратора** — того, чей `TELEGRAM_ADMIN_ID` указан в `.env`. Все остальные пользователи получат отказ.

### Шаг 1 — Создай бота через BotFather

1. Открой [@BotFather](https://t.me/BotFather) в Telegram
2. Отправь команду `/newbot`
3. Введи **имя** бота — любое, например: `Modeus Calendar`
4. Введи **username** бота — латиницей, оканчивается на `bot`, например: `my_modeus_admin_bot`
5. BotFather пришлёт токен вида:
   ```
   1234567890:ABCDEFghijklmnopqrstuvwxyz
   ```
   Это и есть `TELEGRAM_BOT_TOKEN` — скопируй его в `.env`.

### Шаг 2 — Узнай свой Telegram ID

Напиши боту [@userinfobot](https://t.me/userinfobot) — он ответит примерно так:
```
Id: 123456789
First: Иван
```
Число `123456789` — это твой `TELEGRAM_ADMIN_ID`, вставь его в `.env`.

### Шаг 3 — Запусти и открой бота

После `docker compose up -d --build` найди своего бота в Telegram и отправь `/start`. Появится главное меню с кнопками.

## Подключение к Caddy

Если на сервере уже работает Caddy, добавь один блок в `Caddyfile`:

```
your-domain.com {
    reverse_proxy localhost:3000
}
```

Перезагрузи конфиг:

```bash
# Если Caddy запущен как systemd-сервис:
systemctl reload caddy

# Или вручную:
caddy reload --config /etc/caddy/Caddyfile
```

SSL-сертификат выпустится автоматически. Больше ничего не нужно.

## Локальный запуск без Docker

Для разработки можно запустить без Docker.

### Требования

- Node.js >= 18
- npm

### Установка

```bash
git clone <repo-url>
cd modeus-calendar

# Зависимости бэкенда
npm install

# Зависимости фронтенда
cd client && npm install && cd ..

# Зависимости бота (опционально)
cd bot && npm install && cd ..

# Создай .env
cp .env.example .env
nano .env   # заполни MODEUS_USERNAME, MODEUS_PASSWORD и остальные поля
```

### Запуск в режиме разработки

```bash
# Терминал 1 — бэкенд
npm run dev

# Терминал 2 — фронтенд (Vite с hot reload)
npm run dev:client
# → http://localhost:5173 (API автоматически проксируется на :3000)

# Терминал 3 — бот (опционально)
cd bot && npm run dev
```

### Сборка для продакшна (без Docker)

```bash
npm run build    # компилирует TypeScript + собирает React
npm start        # один процесс на одном порту
```

## API

### `POST /api/calendar/register`

Создать подписку по ФИО и инвайт-коду.

**Лимит:** 20 запросов за 15 минут с одного IP.

**Тело запроса:**

| Поле         | Тип    | Описание                                             |
|--------------|--------|------------------------------------------------------|
| `fio`        | string | ФИО (Фамилия Имя Отчество, кириллица)                |
| `inviteCode` | string | Инвайт-код                                           |
| `personId`   | string | (опц.) ID из Modeus — для подтверждения выбора       |
| `personName` | string | (опц.) Полное имя выбранного человека                |

**Пример — единственное совпадение:**

```bash
curl -X POST http://localhost:3000/api/calendar/register \
  -H "Content-Type: application/json" \
  -d '{"fio": "Иванов Иван Иванович", "inviteCode": "A1B2C3D4"}'
```

```json
{
  "message": "Подписка создана для \"Иванов Иван Иванович\".",
  "token": "a3f8c2d1e4b59f7a...",
  "url": "http://localhost:3000/a3f8c2d1e4b59f7a..."
}
```

**Пример — несколько совпадений (выбор):**

```json
{
  "status": "multiple",
  "persons": [
    { "id": "uuid-1", "fullName": "Иванов Иван Иванович", "specialtyName": "Программная инженерия" },
    { "id": "uuid-2", "fullName": "Иванов Иван Иванович", "specialtyName": "Прикладная математика" }
  ]
}
```

После этого фронтенд показывает всплывающее окно выбора и повторяет запрос с `personId` + `personName`.

**Коды ответа:**

| HTTP | Причина                                               |
|------|-------------------------------------------------------|
| 201  | Подписка создана                                      |
| 200  | Подписка уже существует (возвращается та же ссылка)  |
| 200  | Найдено несколько человек (`status: "multiple"`)      |
| 400  | ФИО не прошло валидацию                              |
| 403  | Инвайт-код неверный или уже использован              |
| 404  | Человек не найден в Modeus                           |
| 429  | Превышен лимит запросов                              |

---

### `GET /<token>` или `GET /<token>.ics`

Отдаёт ICS-файл расписания (`text/calendar`). Данные из кэша — Modeus API не вызывается при каждом запросе.

```bash
curl https://your-domain.com/a3f8c2d1e4b59f7a...
```

---

### `POST /api/internal/register` (только для бота)

Регистрация без инвайт-кода. Доступно только с заголовком `X-Internal-Secret`.

```bash
curl -X POST http://localhost:3000/api/internal/register \
  -H "Content-Type: application/json" \
  -H "X-Internal-Secret: твой-INTERNAL_SECRET" \
  -d '{"fio": "Иванов Иван Иванович"}'
```

### 🎟️ Создать инвайт-код

Генерирует одноразовый 8-символьный код (например `A3F8C2D1`) и сразу показывает его:

```
🎟️ Новый инвайт-код создан

Код: A3F8C2D1

Передайте этот код пользователю. Код одноразовый.
```

Передай код пользователю любым способом (в чате, по почте, на бумажке). Пользователь вводит его на сайте вместе с ФИО — после этого код сгорает и больше не работает.

Кнопка **🎟️ Ещё один код** позволяет сгенерировать следующий, не возвращаясь в меню.

### ➕ Добавить пользователя

Позволяет зарегистрировать человека **без инвайт-кода** прямо из бота. Удобно, когда нужно добавить кого-то быстро или у человека нет доступа к сайту.

**Сценарий — один результат:**

1. Нажми кнопку **➕ Добавить пользователя**
2. Бот попросит ввести ФИО:
   ```
   ➕ Добавить пользователя

   Введите полное ФИО пользователя:
   (Фамилия Имя Отчество)
   ```
3. Напиши ФИО текстом: `Иванов Иван Иванович`
4. Бот ищет в Modeus, затем отвечает:
   ```
   ✅ Подписка создана для "Иванов Иван Иванович".

   Ссылка на календарь:
   https://your-domain.com/a3f8c2d1e4b59f7a...
   ```
5. Скопируй ссылку и передай пользователю.

**Сценарий — несколько совпадений:**

Если в Modeus несколько людей с одинаковым ФИО, бот покажет список с кнопками выбора:

```
Найдено несколько человек. Выберите нужного:

[ Иванов Иван Иванович (Программная инженерия) ]
[ Иванов Иван Иванович (Прикладная математика) ]
[ ❌ Отмена ]
```

Нажми нужную кнопку — бот зарегистрирует именно этого человека и вернёт ссылку.

**Сценарий — человек уже зарегистрирован:**

Если подписка для этого человека уже существует, бот просто вернёт ту же ссылку:

```
✅ Подписка уже существует для "Иванов Иван Иванович".

Ссылка на календарь:
https://your-domain.com/a3f8c2d1e4b59f7a...
```

## Возможные проблемы

**Бот не отвечает**

```bash
docker compose logs -f bot
```
Проверь, что `TELEGRAM_BOT_TOKEN` и `TELEGRAM_ADMIN_ID` верно указаны в `.env`.

**`401 Unauthorized` в логах backend**

Токен Modeus истёк (~24 часа). Перезапусти бэкенд — при старте он получит новый:
```bash
docker compose restart backend
```

**`503` при открытии ICS-ссылки**

Первая синхронизация ещё не завершилась. Подожди 30–60 секунд и обнови страницу.

**`Инвайт-код неверный`**

- Коды вводятся заглавными буквами (форма переводит автоматически)
- Каждый код одноразовый — создай новый через бота
- Убедись, что код не был использован ранее (проверь в боте через 🔑)

**`Человек не найден в Modeus`**

- Проверь правильность написания ФИО (Фамилия Имя Отчество полностью, с заглавных букв)
- Убедись, что аккаунт Modeus активен и данные в `.env` верны

**Docker-билд падает с ошибкой компиляции**

`better-sqlite3` компилируется из исходников. Убедись, что на сервере не менее 512 MB RAM — компиляция C++ требует памяти.

```bash
# Если мало памяти — создай swap (1 GB):
dd if=/dev/zero of=/swapfile bs=1M count=1024
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
```

**Расписание не обновляется**

Посмотри логи синхронизации:
```bash
docker compose logs backend | grep -i sync
```

Cron-синхронизация запускается по расписанию `CRON_SCHEDULE` (по умолчанию раз в 3 часа). При старте контейнера синхронизация запускается сразу для всех подписчиков.
