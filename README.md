# Modeus Calendar

Сервис подписки на расписание ТюмГУ в формате iCalendar.

Пользователь вводит ФИО и инвайт-код → получает личную ссылку → добавляет в Apple Calendar / Google Calendar → расписание обновляется автоматически каждые 3 часа.

## Как это работает

1. Администратор создаёт инвайт-коды через Telegram-бота
2. Пользователь открывает сайт, вводит ФИО и инвайт-код (и опционально свой Telegram ID)
3. Если найдено несколько человек с таким ФИО — появляется окно выбора с направлениями
4. Сервис создаёт подписку и выдаёт уникальную ICS-ссылку
5. Пользователь добавляет ссылку в Apple Calendar или Google Calendar
6. Расписание загружается из Modeus и обновляется автоматически каждые 3 часа
7. При изменениях в расписании пользователь получает уведомление в Telegram (если привязал ID)

Ссылки **детерминированы**: один и тот же человек всегда получает одну и ту же ссылку, пока не меняется `INTERNAL_SECRET`. Это значит, что при переезде на новый сервер или восстановлении из бэкапа ссылки остаются рабочими.

## Архитектура

```
Браузер / Apple Calendar / Google Calendar
         │
         ▼
    frontend (nginx:80)
         │
         ├── /api/*    ──────► backend (Node.js :3000)
         └── /<token>  ──────►    │
                                  ├── SQLite (volume calendar-data)
                                  └── cron → Modeus API (utmn.modeus.org)

    bot (Node.js)
         ├── читает SQLite напрямую (статистика, списки, бэкапы)
         ├── /api/internal/register ──► backend (внутренняя сеть Docker)
         └── Bot API ──► уведомляет пользователей при изменениях расписания
```

**Три Docker-контейнера:**

| Контейнер  | Образ        | Назначение                                         |
|------------|--------------|----------------------------------------------------|
| `frontend` | nginx:alpine | React-билд + реверс-прокси на backend              |
| `backend`  | node:22-slim | API, ICS-фиды, cron-синхронизация с Modeus         |
| `bot`      | node:22-slim | Telegram-бот для администрирования и уведомлений   |

## Быстрый старт (Docker)

### Требования

- Docker + Docker Compose (Docker Desktop или `docker-ce`)
- Аккаунт на `utmn.modeus.org` (логин в формате `ivanov.ii@s.utmn.ru`)
- Telegram-бот (токен от [@BotFather](https://t.me/BotFather))

### Шаг 1 — Скопируй проект на сервер

```bash
# Через git:
git clone <repo-url> /root/app
cd /root/app

# Или через rsync с локальной машины:
rsync -av \
  --exclude='node_modules' --exclude='client/node_modules' --exclude='bot/node_modules' \
  --exclude='dist' --exclude='client/dist' --exclude='bot/dist' \
  --exclude='.tokens.json' --exclude='*.db' --exclude='.env' \
  ./modeus-calendar/ root@<IP>:/root/app/
```

### Шаг 2 — Создай `.env`

```bash
cp /root/app/.env.example /root/app/.env
nano /root/app/.env
```

Заполни файл (подробнее в разделе [Переменные окружения](#переменные-окружения)):

```env
MODEUS_USERNAME=ivanov.ii@s.utmn.ru
MODEUS_PASSWORD=ВашПароль

PORT=3000
DB_PATH=/app/data/calendar.db
CRON_SCHEDULE=0 */3 * * *
SYNC_WEEKS_AHEAD=4
SYNC_WEEKS_PAST=3

INVITE_REQUIRED=true
INTERNAL_SECRET=придумай-любую-длинную-строку

TELEGRAM_BOT_TOKEN=1234567890:ABCDEFghijklmnop
TELEGRAM_ADMIN_ID=123456789
PUBLIC_URL=https://your-domain.com
```

> **Где взять `TELEGRAM_ADMIN_ID`?**
> Напиши боту [@userinfobot](https://t.me/userinfobot) — он ответит твоим ID.

> **Как создать бота и получить токен?**
> Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → следуй инструкции → скопируй токен.

### Шаг 3 — Запусти

```bash
cd /root/app
docker compose up -d --build
```

После этого:

- Сайт доступен на `http://<IP>:3000` (или по домену, если настроен)
- Telegram-бот активен, напиши ему `/start`

### Полезные команды

```bash
# Посмотреть статус контейнеров
docker compose ps

# Логи всех контейнеров (Ctrl+C для выхода)
docker compose logs -f

# Логи только бэкенда
docker compose logs -f backend

# Логи только бота
docker compose logs -f bot

# Перезапустить после обновления кода
git pull
docker compose up -d --build

# Остановить (данные сохраняются)
docker compose down

# Остановить и удалить все данные (включая базу!)
docker compose down -v
```

> Данные SQLite хранятся в Docker volume `calendar-data` и **не удаляются** при `docker compose down`.

## Telegram-бот

Бот выполняет две роли:

- **Для администратора** — полная панель управления (инвайты, пользователи, бэкапы)
- **Для пользователей** — просмотр своей ссылки на расписание и получение уведомлений об изменениях

### Шаг 1 — Создай бота через BotFather

1. Открой [@BotFather](https://t.me/BotFather) в Telegram
2. Отправь команду `/newbot`
3. Введи **имя** бота — любое, например: `Modeus Calendar`
4. Введи **username** бота — латиницей, оканчивается на `bot`, например: `my_modeus_calendar_bot`
5. BotFather пришлёт токен вида `1234567890:ABCDEFghijklmnopqrstuvwxyz` — скопируй его в `.env`

### Шаг 2 — Узнай свой Telegram ID

Напиши боту [@userinfobot](https://t.me/userinfobot) — он ответит:
```
Id: 123456789
First: Иван
```
Число `123456789` — это `TELEGRAM_ADMIN_ID`, вставь его в `.env`.

### Шаг 3 — Запусти и открой бота

После `docker compose up -d --build` найди своего бота в Telegram и отправь `/start`.

## Подключение к Caddy

Если на сервере уже работает Caddy, добавь один блок в `Caddyfile`:

```
your-domain.com {
    reverse_proxy localhost:3000
}
```

Перезагрузи конфиг:

```bash
systemctl reload caddy
# или:
caddy reload --config /etc/caddy/Caddyfile
```

SSL-сертификат выпустится автоматически. Больше ничего не нужно.

## Локальный запуск без Docker

### Требования

- Node.js >= 18
- npm

### Установка

```bash
git clone <repo-url>
cd modeus-calendar

# Зависимости бэкенда
npm install

# Зависимости фронтенда
cd client && npm install && cd ..

# Зависимости бота (опционально)
cd bot && npm install && cd ..

# Создай .env
cp .env.example .env
nano .env
```

### Запуск в режиме разработки

```bash
# Терминал 1 — бэкенд
npm run dev

# Терминал 2 — фронтенд (Vite с hot reload)
npm run dev:client
# → http://localhost:5173 (API автоматически проксируется на :3000)

# Терминал 3 — бот (опционально)
cd bot && npm run dev
```

### Сборка для продакшна (без Docker)

```bash
npm run build    # компилирует TypeScript + собирает React
npm start        # один процесс на одном порту
```

## Переменные окружения

Все переменные задаются в файле `.env` в корне проекта.

| Переменная           | Обязательная | По умолчанию       | Описание                                                           |
|----------------------|:------------:|--------------------|-------------------------------------------------------------------|
| `MODEUS_USERNAME`    | ✅           | —                  | Логин от аккаунта Modeus (`ivanov.ii@s.utmn.ru`)                  |
| `MODEUS_PASSWORD`    | ✅           | —                  | Пароль от аккаунта Modeus                                         |
| `PORT`               | —            | `3000`             | Порт Express-сервера                                              |
| `DB_PATH`            | —            | `./calendar.db`    | Путь к SQLite-базе (в Docker: `/app/data/calendar.db`)            |
| `CRON_SCHEDULE`      | —            | `0 */3 * * *`      | Расписание cron-синхронизации                                    |
| `SYNC_WEEKS_AHEAD`   | —            | `4`                | Сколько недель вперёд загружать расписание                        |
| `SYNC_WEEKS_PAST`    | —            | `3`                | Сколько недель назад включать (чтобы прошедшие пары не пропадали) |
| `INVITE_REQUIRED`    | —            | `true`             | Требовать инвайт-код (`false` — отключить для разработки)         |
| `INTERNAL_SECRET`    | ✅           | —                  | Секрет для бота и генерации детерминированных токенов             |
| `TELEGRAM_BOT_TOKEN` | ✅           | —                  | Токен бота от @BotFather                                          |
| `TELEGRAM_ADMIN_ID`  | ✅           | —                  | Твой Telegram ID — только ты будешь управлять ботом               |
| `PUBLIC_URL`         | ✅           | —                  | Публичный URL сервиса (`https://your-domain.com`)                 |

> **Важно:** `INTERNAL_SECRET` используется для генерации детерминированных ссылок на календарь. Если сменишь его — все существующие ссылки перестанут работать для новых пользователей (старые в базе остаются). Храни его как пароль.

## API

### `POST /api/calendar/register`

Создать подписку по ФИО и инвайт-коду.

**Лимит:** 20 запросов за 15 минут с одного IP. Инвайт-код проверяется **до** обращения к Modeus — защита от спама запросами.

**Тело запроса:**

| Поле         | Тип    | Описание                                                   |
|--------------|--------|------------------------------------------------------------|
| `fio`        | string | ФИО (Фамилия Имя Отчество, кириллица)                     |
| `inviteCode` | string | Инвайт-код                                                 |
| `telegramId` | string | (опц.) Telegram ID для уведомлений (только цифры, 5–15)   |
| `personId`   | string | (опц.) ID из Modeus — для подтверждения выбора из списка  |
| `personName` | string | (опц.) Полное имя выбранного человека                     |

**Пример — единственное совпадение:**

```bash
curl -X POST http://localhost:3000/api/calendar/register \
  -H "Content-Type: application/json" \
  -d '{"fio": "Иванов Иван Иванович", "inviteCode": "A1B2C3D4"}'
```

```json
{
  "message": "Подписка создана для \"Иванов Иван Иванович\".",
  "token": "a3f8c2d1e4b59f7a...",
  "url": "https://your-domain.com/a3f8c2d1e4b59f7a..."
}
```

**Пример — несколько совпадений:**

```json
{
  "status": "multiple",
  "persons": [
    { "id": "uuid-1", "fullName": "Иванов Иван Иванович", "specialtyName": "Программная инженерия" },
    { "id": "uuid-2", "fullName": "Иванов Иван Иванович", "specialtyName": "Прикладная математика" }
  ]
}
```

После этого фронтенд показывает всплывающее окно выбора и повторяет запрос с `personId` + `personName`.

**Коды ответа:**

| HTTP | Причина                                              |
|------|------------------------------------------------------|
| 201  | Подписка создана                                     |
| 200  | Найдено несколько человек (`status: "multiple"`)     |
| 400  | ФИО не прошло валидацию                             |
| 403  | Инвайт-код неверный или уже использован             |
| 404  | Человек не найден в Modeus                          |
| 409  | Пользователь уже зарегистрирован                    |
| 429  | Превышен лимит запросов                             |

---

### `GET /<token>` или `GET /<token>.ics`

Отдаёт ICS-файл расписания (`text/calendar`). Данные из кэша — Modeus API не вызывается при каждом запросе.

```bash
curl https://your-domain.com/a3f8c2d1e4b59f7a...
```

---

### `POST /api/internal/register` (только для бота)

Регистрация без инвайт-кода. Доступно только с заголовком `X-Internal-Secret`.

```bash
curl -X POST http://localhost:3000/api/internal/register \
  -H "Content-Type: application/json" \
  -H "X-Internal-Secret: твой-INTERNAL_SECRET" \
  -d '{"fio": "Иванов Иван Иванович"}'
```

### Роль администратора

Если `TELEGRAM_ADMIN_ID` совпадает с твоим ID, бот показывает панель управления.


Вернуться в главное меню можно кнопкой **◀️ Главное меню** или командой `/menu`.

#### 👥 Пользователи (подраздел)

Нажми **👥 Пользователи** — откроется подменю:

| Кнопка              | Действие                                                         |
|---------------------|------------------------------------------------------------------|
| ➕ Добавить         | Зарегистрировать человека без инвайт-кода (поиск по ФИО)        |
| 👥 Список           | Все подписчики с пагинацией (🔔 = привязан Telegram)             |
| 🔗 Найти ссылку     | Найти ссылку на календарь по части ФИО                          |
| 🔗 Привязать TG ID  | Привязать Telegram ID к существующей подписке вручную           |
| 🗑️ Удалить          | Удалить подписку и кэш расписания пользователя                  |

**Добавить пользователя:**

1. Нажми ➕ Добавить
2. Введи ФИО текстом
3. Бот ищет в Modeus и регистрирует (или показывает список для выбора)
4. Ссылка возвращается сразу

**Привязать TG ID:**

1. Нажми 🔗 Привязать TG ID
2. Введи ФИО (или часть)
3. Выбери пользователя из списка
4. Введи числовой Telegram ID
5. Пользователь начнёт получать уведомления при изменениях расписания

**Удалить пользователя:**

1. Нажми 🗑️ Удалить
2. Введи ФИО (или часть)
3. Подтверди удаление — действие необратимо

#### 🎟️ Инвайты (подраздел)

| Кнопка           | Действие                                              |
|------------------|-------------------------------------------------------|
| 🎟️ Создать код   | Генерирует одноразовый 8-символьный код               |
| 🔑 Список кодов  | Все коды с пагинацией, кнопка удаления для свободных  |

Коды одноразовые — сгорают при использовании. Неиспользованные можно удалить прямо из списка.

#### 📦 Бэкапы

| Кнопка                     | Действие                                                        |
|----------------------------|-----------------------------------------------------------------|
| 💾 Создать и отправить     | Создаёт резервную копию SQLite и присылает файл в чат          |

```bash

```

### Роль пользователя

Любой человек может написать боту `/start`. Если его Telegram ID привязан к подписке — бот покажет ссылку на расписание. Если нет — покажет ID, который нужно передать администратору.

**Уведомления об изменениях:**

При каждой синхронизации бот сравнивает старое и новое расписание. Если обнаружены изменения в будущих занятиях — пользователь получает сообщение вида:

```
📅 Изменения в расписании

➕ Добавлено:
• Математический анализ — 15 мар, 10:00–11:30

🗑 Удалено:
• Химия — 16 мар, 15:00–16:30

✏️ Изменено:
• Программирование — 14 мар, 09:00–10:30
```

Привязать Telegram ID можно:
- При регистрации на сайте (поле «Telegram ID», необязательное)
- Через администратора (кнопка 🔗 Привязать TG ID в боте)

## Возможные проблемы

**Бот не отвечает**

```bash
docker compose logs -f bot
```
Проверь, что `TELEGRAM_BOT_TOKEN` и `TELEGRAM_ADMIN_ID` верно указаны в `.env`.

**`401 Unauthorized` в логах backend**

Токен Modeus истёк (~24 часа). Перезапусти бэкенд — при старте он получит новый:
```bash
docker compose restart backend
```
При каждой ошибке авторизации бот автоматически уведомит администратора в Telegram.

**`503` при открытии ICS-ссылки**

Первая синхронизация ещё не завершилась. Подожди 30–60 секунд и обнови страницу.

**`Инвайт-код неверный`**

- Коды вводятся заглавными буквами (форма переводит автоматически)
- Каждый код одноразовый — создай новый через бота
- Убедись, что код не был использован ранее (проверь через 🔑 Список кодов)

**`Человек не найден в Modeus`**

- Проверь правильность ФИО (Фамилия Имя Отчество полностью, с заглавных букв)
- Убедись, что аккаунт Modeus активен и данные в `.env` верны

**`Пользователь уже зарегистрирован`**

Каждый человек может иметь только одну подписку. Найти существующую ссылку можно через 🔗 Найти ссылку в боте.

**Расписание не обновляется**

```bash
docker compose logs backend | grep -i sync
```

Cron-синхронизация запускается по расписанию `CRON_SCHEDULE` (по умолчанию раз в 3 часа). При старте контейнера синхронизация запускается немедленно.

**Docker-билд падает с ошибкой**

Если мало памяти (< 512 MB) — создай swap:
```bash
dd if=/dev/zero of=/swapfile bs=1M count=1024
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
```
